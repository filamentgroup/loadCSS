CSS Loading Diagnostic Tests

This directory contains a variety of files that employ various CSS loading techniques. Most files reference a CSS file that is generated by PHP to include a server delay to mimic latency. The delay makes it easier to determine if content is rendered before the CSS loads.

Standard Loading

  • Link in Head: Does a link to a stylesheet from the head of the page block page rendering?
  • Link in Body: Does a stylesheet link in the body of the page block all page rendering, rendering of content that comes after it, or not block at all?
  • @import in Head: Does an inline @import reference from the head of the page block rendering?
  • @import in Body: Does an inline @import reference from the body of the page block rendering of content below it?
  • DOM appendChild: Create a link element and append to the document.

link[rel=preload]

  • Rel=preload Test: standard link[rel=preload] with an onload handler to apply as a stylesheet once loaded. This will only work in supporting browsers.

  • Polyfill for modern browsers (IE9+): Stylesheet preloaded and polyfilled if necessary

    • Polyfill after style link tags: The recommended setup, all links will be found immediately
    • Polyfill before style link tags: Polyfill will get links that come late3r but with a delay

  • Polyfill for legacy browsers (IE6+): Stylesheet preloaded and polyfilled if necessary

    • Polyfill after style link tags: The recommended setup, all links will be found immediately
    • Polyfill before style link tags: Polyfill will get links that come late3r but with a delay

loadCSS Assisted Loading

  • loadCSS Dynamic Load: Stylesheet loaded with loadCSS() dynamically.
  • loadCSS Dynamic Load with onload handler: Same as above with onload script handler.
  • loadCSS Dynamic Load with insertion settings: Stylesheet loaded with loadCSS() and inserted as per given options.
  • Link attributes: Do browsers supporting Subresource Integrity correctly handle link attributes?
